# Change: 改造 ping 接口为基于时间的动态响应

## 为什么（Why）
当前 `/ping` 接口无论何时调用都固定返回 `pong`，缺乏时间感知能力。需要根据业务时段返回不同响应，以区分服务的工作时段与非工作时段。

## 变更内容（What Changes）
- `/ping` 接口在 10:00–18:00（含 10:00，不含 18:00）返回 `pong`
- `/ping` 接口在其他时段返回 `pang`
- 时间判断基于服务器本地时间

## 影响范围（Impact）
- 受影响模块：`main.go` 中的 `/ping` handler
- 无外部依赖变更
- 现有调用方若依赖固定返回 `pong` 将感知到行为变化
