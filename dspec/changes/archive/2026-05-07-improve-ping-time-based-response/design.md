# Design: 改造 ping 接口为基于时间的动态响应

## 技术方案
在 `main.go` 的 `/ping` handler 中，使用 `time.Now()` 获取当前本地时间，提取小时数，根据是否在 [10, 18) 区间内决定响应内容。

选择直接内联时间判断逻辑，不抽取独立函数——改动仅一处，逻辑简单，过度抽象无必要。

## 模块结构
- `main.go`：修改 `/ping` handler，添加时间判断分支

## 关键决策
- 时间边界：10:00 含、18:00 不含，即 `hour >= 10 && hour < 18`
- 使用服务器本地时间（`time.Now()`），不引入时区配置
- 响应格式保持原有纯文本 `fmt.Fprintln`，不改为 JSON
