# 提案：为所有接口添加 API 文档

## 背景
当前项目有 10 个 HTTP 接口，但没有统一的接口文档。需要为所有接口编写标准的 API 文档，方便调用方理解和使用。

## 目标
为以下 10 个接口生成完整的 API 文档：
1. GET / - 首页
2. GET /ping - 健康检查
3. GET /test - 测试接口
4. POST /add - 字符串拼接
5. POST /variance - 方差计算
6. GET /random - 随机数生成
7. GET /split - 字符串分割
8. GET /timestamp - 时间戳查询
9. GET /timezone - 时区查询
10. POST /md5 - MD5 哈希计算

## 方案
在 `specs/api/` 目录下创建 `spec.md`，按标准格式记录每个接口的：
- 请求方式和地址
- 请求参数
- 响应字段
- 响应示例
- 错误示例

## 范围
- 仅生成文档，不修改代码
- 文档格式统一，便于维护

## 验收标准
- 所有 10 个接口均有文档
- 文档格式符合团队规范
