# Design: 添加 SHA1 哈希计算端点

## 技术方案
采用与现有 MD5、SHA256 端点一致的实现模式：
- 使用 Go 标准库 `crypto/sha1` 进行哈希计算
- 复用现有的请求/响应结构模式
- 遵循统一的错误处理约定

## 模块结构

### main.go
新增以下内容：
- `Sha1Request` 结构体：请求参数，包含 `input` 字段
- `Sha1Response` 结构体：响应数据，包含 `hash` 字段
- `sha1Handler` 函数：处理 `POST /sha1` 请求

### 路由注册
在 `main()` 函数中注册新路由：
```go
mux.HandleFunc("/sha1", sha1Handler)
```

## 关键决策

### 1. 哈希输出格式
选择小写十六进制输出，与 MD5、SHA256 保持一致，符合 Go `hex.EncodeToString` 的默认行为。

### 2. 空 input 处理
与 SHA256 行为一致：当 `input` 字段缺失或为空字符串时，返回对应输入的哈希值而非错误。

### 3. 错误处理
仅对无效 JSON 返回 400 错误，其他情况返回正常哈希值。

### 4. 安全提示
代码注释中说明 SHA1 不适用于安全敏感场景（如密码存储、数字签名），仅用于兼容性和校验场景。
