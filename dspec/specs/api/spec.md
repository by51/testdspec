## 13. URL 解码

### 接口说明
对 URL 编码的字符串进行解码。

### 请求方式
`POST`

### 请求地址
`/urldecode`

### 请求参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| input | string | 否 | 待解码的 URL 编码字符串，默认为空 |

### 响应字段
| 字段 | 类型 | 说明 |
|------|------|------|
| decoded | string | 解码后的字符串 |

### 响应示例
```json
{
  "decoded": "hello world"
}
```

### 错误示例
```json
{
  "error": "invalid URL encoding"
}
```

### 场景说明
- 正常解码 URL 编码字符串（如 `hello%20world` → `hello world`）
- 解码中文字符（如 `%E4%BD%A0%E5%A5%BD` → `你好`）
- 解码特殊字符（如 `a%3Db%26c%3Dd` → `a=b&c=d`）
- 解码未编码字符串（直接返回原字符串）
- 解码空字符串（返回空字符串）
- 无效 URL 编码格式返回 400 错误（如 `%ZZ`）
- 百分号后缺少两位十六进制字符返回 400 错误（如 `hello%2`）
- 缺少 input 字段时默认为空字符串
- 无效 JSON 格式返回 400 错误
- 非 POST 方法返回 405 错误
